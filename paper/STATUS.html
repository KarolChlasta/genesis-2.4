Key findings
Technical record
F1
GENESIS scheduler is the dominant bottleneck. Iterates all absorbed elements every step regardless of GPU offload: ~25–45 ns/element/step, O(N). Accounts for 98–99% of GPU arm step time at N≥2000.
F2
Multiloop mode achieves 3.2–16.2× end-to-end speedup after scheduler fix + fp32 port. Raw kernel speedup: 4–149×. GPU per-step remains slower than CPU.
F3
Three hines_init.c/hines_solve.c bugs silently broke all multi-neuron hsolve. SKIP_DIAG OOB (Bug 3), missing parentno==-1 guard (Bug 2), wrong ncompts==1 proxy (Bug 1) + euler stray −vm term. Classic single-neuron usage never hit any of them.
F4
AMD Radeon 890M has zero fp64 OpenCL support. Kernel must run in float. Host converts double↔float at upload/download time in ocl_hsolve.c.
F5
All three execution paths verified to fp32 precision. CPU, GPU per-step, GPU multiloop agree within 3×10⁻⁹ V after one 50 µs HH step with 2 nA injection, N=1 and N=2.
F6
OCL ICD vendor path is critical. Wrong path → silent CPU fallback. Correct: /run/host/etc/OpenCL/vendors.
F7
PGENESIS MPI strong-scaling saturates at 12 physical cores. N=2400 HH1952 neurons, 5000 steps: P=2 S=1.89× (E=95%), P=8 S=4.0× (E=50%), P=12 S=4.5× (peak). Beyond P=12 SMT provides no benefit. Recommended: P=4–8 for this problem size.
